# Open-access Paper Analysis with GROBID

Repository for the **Artificial Intelligence and Open Science** assignment.  
This project analyzes **10 open-access scientific articles** using **GROBID** and complementary PDF parsing.

## Objectives

The program performs the following tasks:

1. Extracts the **abstract** of each paper and generates a **keyword cloud**
2. Creates a **visualization of the number of figures per article**
3. Produces a **list of links found in each paper**

# Validation of results
1. Keyword cloud validation

The keyword cloud is generated only from the abstracts extracted from the GROBID TEI XML output.
Validation method:

I manually checked that each TEI file contains an <abstract> section.

I compared a sample of extracted abstract text against the original PDFs.

The cloud is generated from the concatenation of all extracted abstracts, excluding common stopwords.

2. Figures per article validation

The number of figures is estimated from GROBID TEI <figure> elements, using figure labels/captions such as Figure 1 or Fig. 2.
Validation method:

I manually inspected several TEI files and compared them with the original PDFs.

I verified that figure-like labels were counted and non-figure labels such as Algorithm 1 were not counted as figures.

The resulting counts were exported to results/figures_per_article.csv and visualized in results/figures_chart.png.

3. Links per paper validation

Links are extracted from two sources:

TEI references generated by GROBID

PDF hyperlink annotations extracted directly from the original PDF

# Validation method:

I compared the extracted links of several papers against the original PDF.

I checked that duplicated links were removed.

I verified that the final output is grouped by paper in results/links_per_paper.json.
